# Reference Concept Analysis — Post-apply assessment

- **Target project:** Codepatrol v1.0 (local, harness-agnostic CLI + skill tree + artifact handoff)
- **Target problem:** identify market concepts that would improve v1.0 without violating the local-only / no-scheduler / no-MCP product contract.
- **Reference:** Microsoft `agent-skills` (commit `d884ae04edebef577e82ff7c4e143debd0bbec99` was a primary source consulted in the 2026-07-21-apply-orchestration-hardening analysis; this assessment re-reads only what is new), Phoenixrr2113/agent-harness/docs/skill-evals.md, OpenAI Agents SDK handoffs and tracing docs, MCP `2025-06-18` + `2026-07-28` release candidate, AWS Bedrock AgentCore Memory SDK, GitHub Copilot Coding Agent risks and mitigations, MITRE ATLAS AI agent threat model.
- **Revision:** consulted 2026-07-21; the external sources are continuously updated. `agent-skills` and the SDKs each have a recent commit; MCP `2026-07-28` is a release candidate, not a final spec.
- **Consulted:** 2026-07-21
- **Scope:** read the docs and the high-level architecture; do not run, install, or import anything.

## Local constraints

Codepatrol's product contract (per `README.md`): "The repository has no MCP server and no agent scheduler. Skills describe independent work units and a synchronization barrier in natural language. Harnesses with native delegation may run those units in parallel; Pi and restricted harness sessions run the same units sequentially." There is no hosted telemetry, no `subagent` tool, no harness-specific code outside the `.pi/`, `.claude/`, `.codex/`, `.opencode/` dotfolders. The catalog enforces primary order (1..4), reciprocal `mayInvoke` ⇄ `invokedBy`, support-only trigger targets, and a finite `when` value set.

## Recommendation

Adopt the executable skill-evaluation harness as the bounded upgrade path for DC-1. Adapt the per-task guardrail concept (already done in T3 at the latest commit). Reject direct integrations of MCP, OpenAI Agents SDK, AWS Bedrock AgentCore Memory SDK, GitHub Copilot Coding Agent, and MITRE ATLAS. Document the rejected integration surfaces in `CONTEXT.md` to prevent future re-litigation.
