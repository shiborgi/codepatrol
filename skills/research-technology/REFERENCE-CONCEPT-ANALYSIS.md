# Reference Concept Analysis format

```markdown
# Reference Concept Analysis — <topic>

- **Target project:** <project and bounded context>
- **Target problem:** <problem to solve>
- **Reference:** <canonical URL>
- **Revision:** <tag or commit>
- **Consulted:** <YYYY-MM-DD>
- **Scope:** <examined and excluded material>

## Local constraints
<Settled constraints and local evidence.>

## Observed concepts

### <concept>
- **Facts:** <cited observations>
- **Inference:** <how the mechanism works and uncertainty>
- **Reference problem:** <problem solved there>
- **Target fit:** <same and different constraints>
- **Decision:** `adopt | adapt | reject`
- **Native adaptation:** <target-domain design, or not applicable>
- **Trade-offs:** <benefits, costs, risks>
- **Verification needed:** <local checks>

## Rejected integration surfaces
<Dependencies, adapters, compatible formats, or copied mechanisms deliberately excluded.>

## Recommendation
<What the target project should do and why.>

## Open decisions
<Only choices the calling workflow or user must settle.>
```
